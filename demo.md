# Demo — EdgeOne Pages Full-Stack Site Skill

This demo walks through how an AI Agent uses the **edgeone-pages-fullstack-site** Skill to turn one sentence into a working, deployed commercial website.

---

## 0. Prerequisites

- Node.js ≥ 18
- An AI tool with Skills support (Claude Code, CodeBuddy, Cursor, etc.)
- (Optional, for deploy) An [EdgeOne Pages](https://edgeone.ai/products/pages) account
- (Optional, per module)
  - **Pay:** Stripe test keys
  - **AI Agent:** an OpenAI-compatible API key (OpenAI / DeepSeek / Claude …)

Install the skill:

```bash
# Claude Code
git clone https://github.com/Ilovecodinghhh/edgeone-fullstack-site-skill.git
cp -r edgeone-fullstack-site-skill/skills/edgeone-pages-fullstack-site/ ~/.claude/skills/
```

Or just tell your agent:

> Please install this skill: `https://github.com/Ilovecodinghhh/edgeone-fullstack-site-skill`

---

## 1. Trigger the Skill (one sentence)

In any AI conversation:

> **Build me an e-commerce site with login, Stripe checkout, and an AI customer-service assistant. Deploy to EdgeOne Pages.**

The agent auto-loads `SKILL.md` and runs the decision flow.

---

## 2. Module Selection

The agent confirms which modules to scaffold:

| Module | Default | Notes |
|---|---|---|
| 🔐 Auth | ✅ | JWT + KV sessions (Edge Functions) |
| 🛒 Commerce | ✅ | Cart + inventory + order state machine |
| 💳 Pay | ✅ | Stripe (Alipay / WechatPay stubs ready) |
| 🤖 AI Agent | ✅ | LLM proxy + RAG + streaming |
| 📊 Admin | ✅ | Dashboard + config + user mgmt |

Or pick a template: 🛒 *E-commerce*, 🔧 *SaaS*, 📝 *Content + Paywall*, 🎯 *Custom*.

---

## 3. Project Scaffolding

The skill generates:

```
my-shop/
├── src/                  # React 18 + Vite + TS + Tailwind
│   ├── pages/            # Home / Product / Cart / Checkout / Admin / Chat
│   ├── components/
│   └── lib/api.ts
├── functions/
│   ├── edge/             # auth, session, middleware (V8)
│   └── node/             # commerce, pay, ai, admin (Express)
├── middleware/           # CORS + JWT guard + rate limit
├── kv/                   # KV key contracts
└── edgeone.json
```

Install & run locally:

```bash
cd my-shop
npm install
npm run dev          # Vite frontend
npx edgeone dev      # Edge + Cloud functions emulator
```

Visit `http://localhost:5173`.

---

## 4. Configure Environment

The skill prompts for `.env` values and explains where to obtain each:

```env
JWT_SECRET=<auto-generated>
STRIPE_SECRET_KEY=sk_test_...
STRIPE_WEBHOOK_SECRET=whsec_...
LLM_API_KEY=sk-...
LLM_BASE_URL=https://api.openai.com/v1
LLM_MODEL=gpt-4o-mini
EDGEONE_KV_NAMESPACE=my-shop-kv
```

See `references/env-setup.md` for every key.

---

## 5. Smoke Test the Flow

| Step | Action | Expected |
|---|---|---|
| 1 | Register a user at `/signup` | 201 + JWT cookie |
| 2 | Add product → cart | Cart count updates, inventory locked |
| 3 | `/checkout` → Stripe test card `4242 4242 4242 4242` | `payment_succeeded` webhook → order = `paid` |
| 4 | Click chat bubble, ask *"Where is my order?"* | AI replies with live order status (intent → action bridge) |
| 5 | Log in to `/admin` | Revenue chart, orders table, config hot-reload |

---

## 6. Deploy

Hand off to the official deploy skill:

> Deploy this project to EdgeOne Pages.

The companion `edgeone-pages-deploy` skill takes over:

```bash
npx edgeone deploy
```

You get a live URL like `https://my-shop.edgeone.app` — complete with edge auth, Stripe checkout, and an AI assistant.

---

## 7. What to Try Next

- Swap the LLM provider (DeepSeek / Claude) — change `LLM_BASE_URL` only.
- Add a new product category — edit `kv/products.ts`, hot-reload via admin.
- Enable Alipay — fill the stub in `functions/node/pay/alipay.ts`.
- Plug in your own RAG corpus — drop markdown into `references/kb/`.

---

**Total time, blank repo → live site: ~10 minutes.**
